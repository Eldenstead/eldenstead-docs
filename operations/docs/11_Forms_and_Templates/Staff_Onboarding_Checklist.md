# New Staff Setup Checklist

Use this checklist to make sure a new staff member receives the right training and only the access they need.

## Identity and appointment

- [ ] Display name, Minecraft name/UUID, and Discord account recorded
- [ ] Role, team lead, start date, and probation end date recorded
- [ ] Paid/voluntary terms accurately documented where applicable
- [ ] Conflict disclosures completed

## Required reading

- [ ] GOV-001 Roles and Authority
- [ ] GOV-003 Conduct and Confidentiality
- [ ] SEC-001 Account Security
- [ ] SEC-002 Secrets and Login Details
- [ ] Role-specific SOPs assigned
- [ ] Unannounced-content expectations acknowledged

## Account security

- [ ] Unique password/password manager
- [ ] MFA enabled
- [ ] Recovery methods secured
- [ ] No shared login details provided

## Access

| Service | Role/access | Approver | Date | Expiration |
|---|---|---|---|---|
| | | | | |

- [ ] Access tested for allowed actions
- [ ] Denied actions confirmed where practical
- [ ] Access inventory updated

## Orientation and training

- [ ] Community-first/non-pay-to-win direction explained
- [ ] Staff channels and records explained
- [ ] Escalation and incident process explained
- [ ] Staff action vs normal gameplay/roleplay explained
- [ ] First supervised task assigned
- [ ] Mentor/check-in schedule created
