# Eldenstead Plain-Language Glossary

> Use this page when an SOP includes a staff, moderation, payment, or technical word you do not know.

## A–C

### Appeal
A request to review a punishment or staff decision again.

### Approver
The person who has the power to say yes or no to an action.

### Backup
A saved copy of files or data. Staff can use it to restore the server after damage or loss.

### Canon / official lore
Lore that Eldenstead has approved as an official part of the world and story. Player-created lore is not official unless staff approve it.

### Chargeback / payment dispute
When a buyer asks their bank or payment provider to reverse a payment.

### Compromised account
An account that may be hacked, stolen, or exposed to someone who should not have access.

### Conflict of interest
A personal connection that may make it hard for a staff member to be fair. Examples include reviewing a close friend's punishment or running a war involving your own kingdom.

### Credentials / login details
Passwords, login tokens, recovery codes, and other information used to enter an account.

## D–H

### Data corruption
Damage that makes files, worlds, inventories, or database records wrong or unreadable.

### Deploy / deployment
To place approved code, a plugin, or settings onto the test server or live server.

### Dupe / duplication exploit
A bug or cheat that creates extra copies of items, money, or other value.

### Escalate / escalation
To ask a higher staff role for help, review, or approval.

### Evidence
Information that helps show what happened. Examples include logs, screenshots, video, dates, commands, and test results.

### Exploit
Using a bug or weak part of a system to gain an unfair result or cause harm.

### Hotfix / emergency fix
A small urgent update used to stop a serious live problem.

## I–P

### Incident
A problem that needs staff action, such as an outage, exploit, hacked account, or data loss.

### Incident Lead
The one person chosen to direct staff during a serious incident.

### Least privilege
Giving a person only the access needed for their current job. A Moderator does not need database access just because they are trusted.

### Logs
Automatic records made by the server, website, Discord bot, plugin, or other system.

### MFA / multi-factor authentication
A second login step after the password. It may use an app, passkey, hardware key, or security code.

### Non-pay-to-win
Store purchases may look nice or add harmless features, but they must not give an unfair gameplay advantage.

### Permission node / permission setting
A setting that allows or blocks a command or action for a player or staff role.

### Production / live server
The real server or system used by players. Changes here can affect the community right away.

## R–Z

### Repository
A place where developers save and track code changes, such as a Git repository.

### Rollback
To undo a change or return files and data to an earlier saved state.

### Severity
A label that shows how urgent and harmful a problem is. Eldenstead uses SEV-1 through SEV-4.

### Staging / test server
A separate place used to test changes before they reach the live server.

### Token
A secret string that lets a bot, app, or account connect to a service. Treat it like a password.

### Triage / first review
The first check of a report or bug. Staff decide how serious it is, what information is missing, and who should handle it.

### UUID
A unique ID tied to a Minecraft account. It usually stays the same even when the player changes their name.

### WorldEdit
A powerful building tool that can change many blocks at once. Mistakes can cause large damage, so staff must use backups and limits.

---

## When a word is still unclear

Do not guess when a word changes what you are allowed to do. Ask your team lead or the SOP owner before using a command, giving a punishment, changing data, or promising something to a player.
